## Defaults

- Default project: **DO**
- Default "Work Classification" (custom field, DO space only): **Internal Tools and Processes**

## Ticket titles

Use the format `<repo or project>: <description>`. The description should be
a concise summary of the work, not a list of files or commits.

## Ticket descriptions

Write high-level descriptions of the task and any relevant context. Focus on
the *what* and *why*, not on specific code changes, commits, or file paths.
Keep descriptions useful for someone triaging or prioritizing work, not
reviewing code.
